# コードレビュー

`$ARGUMENTS` を対象にコードレビューを行います。

## レビュー観点

### 1. 型安全性
- `any` の不適切な使用がないか
- Props の型定義が適切か
- null / undefined の考慮が漏れていないか

### 2. コード品質
- DRY 原則に違反していないか
- 関数・変数名が意図を明確に表しているか
- 不要な副作用がないか

### 3. React ベストプラクティス
- Hooks のルールに従っているか
- 不必要な再レンダリングが発生していないか
- コンポーネントの責務が明確か

### 4. セキュリティ
- XSS の危険がないか（`dangerouslySetInnerHTML` 等）
- 機密情報がハードコードされていないか

### 5. パフォーマンス
- N+1 問題や不要な計算がないか
- メモ化（`useMemo`, `useCallback`）が適切に使われているか

## フィードバック形式

各指摘について以下を明記すること：
- **深刻度**: 🔴 必須修正 / 🟡 改善推奨 / 🟢 提案
- **場所**: ファイル名と行番号
- **内容**: 問題点と改善案
